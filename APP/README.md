# APP/ — 应用程序源代码

这个文件夹是"杰昊脚本管理器"桌面应用的源代码。

---

## 文件说明

### 核心运行文件

| 文件 | 干嘛的 |
|------|--------|
| `run_app.py` | 应用程序的入口点。打开 app 实际上就是运行这个文件。包含主界面、运行界面、日志界面的切换逻辑 |
| `ui_main.py` | 主界面长什么样。包括：脚本选择列表、循环次数设置、运行/查看日志/退出按钮 |
| `ui_run.py` | 点"运行"之后看到的界面。包括：实时终端输出、进度显示、截断按钮 |
| `ui_logs.py` | 历史记录界面。按日期分组显示所有运行记录，点击可以查看完整终端输出 |
| `process_manager.py` | 底层进程控制模块。负责启动子进程、读取输出、截断进程 |
| `logger.py` | 日志管理模块。负责保存运行记录到 `~/Library/Logs/杰昊脚本管理器/` |

### 配置和资源

| 文件 | 干嘛的 |
|------|--------|
| `paths.py` | 路径配置文件。把 Setting.py 里的路径配置重新导出给应用内部用 |
| `LOGO/` | 应用图标。包含 `JIEHAO_LOGO.icns`（Mac 应用图标）和 `JIEHAO_LOGO.png`（备用图标） |

### 打包配置

| 文件 | 干嘛的 |
|------|--------|
| `杰昊脚本管理器.spec` | PyInstaller 打包配置文件。如果修改了应用代码或添加了新资源，需要重新打包时用 |

---

## 怎么重新打包

如果你修改了应用代码（比如改了 ui_main.py），需要重新生成 `.app` 文件：

```bash
cd APP
pyinstaller 杰昊脚本管理器.spec
```

打包完成后，新的应用程序会出现在 `APP/dist/杰昊脚本管理器.app`，然后你可以把它拖到 Code/ 根目录下覆盖旧的。

---

## 应用的工作方式

1. 打开 `杰昊脚本管理器.app`
2. 在主界面勾选要运行的脚本（比如 `Compare_JIEHAO.py`），设置循环次数
3. 点"运行"
4. 应用会启动一个子进程执行 Python 脚本
5. 实时输出会显示在运行界面
6. 运行记录会自动保存到日志文件
7. 可以随时点"查看日志"回顾历史记录
