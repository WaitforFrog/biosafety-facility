# 杰昊脚本管理器 - Code 总目录

这是整个项目的大本营，所有代码、资源、生成物都在这里。

---

## 文件夹一览（通俗版）

### 杰昊脚本管理器.app
就是那个双击就能用的 Mac 应用程序！打开它，你会看到一个界面，可以勾选要运行的脚本（比如"生成文章"），设置跑几次，然后点运行就行了。不用懂代码，直接用就行。

### APP/
应用程序的"源代码"。里面的文件负责：
- `run_app.py` — 应用程序的入口，打开 app 其实就是运行这个文件
- `ui_main.py` — 主界面长什么样（脚本选择列表）
- `ui_run.py` — 点运行之后看到的界面（实时输出终端）
- `ui_logs.py` — 查看历史运行记录
- `process_manager.py` — 负责启动和控制脚本进程
- `logger.py` — 记录每次运行的结果
- `paths.py` — 路径配置文件
- `LOGO/` — 应用程序的图标文件
- `杰昊脚本管理器.spec` — PyInstaller 打包配置，打包前需要修改这个文件

### Produce/
文章生成引擎，是整个项目的"大脑"。运行里面的脚本可以让 AI 自动帮你写文章。

### Check/
产品参数文件夹。里面的 markdown 文件记录了每个产品的技术参数，AI 生成文章时会读取这些参数。

### Title/
产品英文标题库。每个产品有一个 txt 文件，里面写着这个产品对应的英文文件夹名（用于 SEO 友好的网址）。

### Website/
AI 生成的网站文章存放的地方。每个产品一个文件夹，里面是静态 HTML 页面，这就是最终要放到服务器上的网站。

### 参数/
存放产品相关的参数数据文件。

### 文章/
生成的文章备份。每次 AI 写完文章，除了放到 Website/，还会在这里留一份备份。`汇总日志/` 里记录每次运行的详细结果（JSON 格式）。

### preview_site/
网站预览相关文件。

### Setting.py + Setting.md
配置文件。这两个文件控制着整个项目的行为：
- `Setting.md` 是给人类看的配置文件，可以用普通文本编辑器修改
- `Setting.py` 负责读取 Setting.md 并提供给其他脚本使用

需要改 API 配置（比如换了个 AI 密钥），直接编辑 Setting.md 就行。

---

## 快速使用

1. 双击 `杰昊脚本管理器.app` 打开应用程序
2. 勾选要运行的脚本（比如 Compare_JIEHAO.py）
3. 设置循环次数（跑几轮）
4. 点"运行"

或者，如果你想用命令行直接跑：
```bash
python Produce/Compare_JIEHAO.py
```
