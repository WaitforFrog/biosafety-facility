# IQ (Installation Qualification) Documentation Preparation and Execution: Protocol, Checklist, and Evidence Collection for Regulatory Submission

## 适用受众
Commissioning_Engineer

## 写作角度说明
IQ验证文件准备与执行是调试工程师为生物安全设备编制并执行安装确认（IQ）文件的关键工作。IQ是设备验证生命周期（IQ/OQ/PQ）的第一步，需要证明设备在现场的安装符合设计规格和法规要求。

## 核心论点 / 洞察
Completing the IQ protocol without referencing the manufacturer design specification — relying solely on generic IQ templates — leaves documented gaps that auditors will flag when the protocol is reviewed against the actual validation master plan.

## 参考素材
- IQ protocol structure: introduction and scope → references (validation master plan, design specifications, FAT records) → IQ items (equipment identification, location, utilities verification, materials verification, spare parts verification) → acceptance criteria → deviation reporting → certification and sign-off
- Key IQ items: equipment identification (model, serial number, manufacturer, year of manufacture) → installation environment verification (temperature, humidity, cleanliness class) → utilities verification (power supply voltage, frequency, air supply pressure) → software/firmware version verification → calibration certificates for test equipment used
- Evidence requirements: for each IQ item → one or more objective evidence documents (photos, test data, certificates, screenshots) linked to specific IQ item
- Deviation management: any IQ item not meeting acceptance criteria → formal deviation report → impact assessment → corrective action → re-test and close
- Regulatory references: GMP Annex 1 (for pharmaceutical cleanrooms), FDA 21 CFR Part 211 (for GMP facilities), EU GMP Annex 11 (for computerized systems)
