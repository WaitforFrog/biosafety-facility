# Installation Change Management Process: Documenting and Approving Field Modifications to Equipment Installation Requirements

## 适用受众
Site_Supervisor

## 写作角度说明
安装变更管理流程是现场主管控制现场变更、确保变更可追溯的核心工具。安装过程中可能因现场条件变化、设计错误或设备到货问题产生变更需求，不受控的变更是质量事故的主要来源。

## 核心论点 / 洞察
Verbal change approvals — communicated through foremen rather than formal documentation — create scope disputes during commissioning that have no resolution mechanism because no written record exists.

## 参考素材
- Change initiation: any deviation from approved installation drawings or specifications must be documented on a formal change request form within 24 hours of identification
- Approval hierarchy: minor changes (affecting single equipment unit, <4 hours work) → site supervisor approval; major changes (affecting multiple systems or schedule) → project manager and client approval
- Documentation update: approved changes must be reflected in as-built drawings within 5 working days of approval, change log updated, all affected stakeholders notified
- Re-commissioning trigger: changes affecting structural integrity, seal configuration, or control logic require re-commissioning of affected system
- Cost and schedule impact: each change request must include estimated cost and schedule impact before approval, changes cannot proceed without approval
