# [验证文件版本管理与受控分发：质量记录的数据完整性保障]
## 适用受众
Validation_Specialist
## 写作角度说明
验证文件（方案、测试记录、报告）是GMP质量记录体系的核心组成部分，其版本控制不善将导致"文档与实际状态不一致"的重大合规风险。本模块覆盖验证文件的版本控制、分发追踪、受控存档和保存期限管理。
## 核心论点 / 洞察
验证文件的版本控制不是"在文件名里加个v1.0/v2.0"这么简单——有效的版本控制需要建立主文件索引（Master File Index），记录每一版文件的变更理由、审批记录和分发去向。当两份不同版本的验证方案同时在用时，这就是一个Critical级偏差。
## 参考素材
- 21 CFR Part 211（FDA GMP）：质量控制记录要求
- EU GMP Chapter 4：Documentation
- ISPE GAMP 5：文件管理要求
- 文件命名规范：文件代号_文件类型_版本号_日期（如：VAL-PASSBOX-001-OQ-v2.0-20240115）
- 主文件索引（MFI/Master File Index）：记录所有受控文件的当前版本号、版本日期、存放位置、分发对象
- 变更控制：文件修订需填写变更申请（Change Control），说明变更理由和影响评估，经审批后方可发布新版本
- 分发追踪：每次文件分发需记录：文件编号/版本号、分发日期、接收人签名、分发份数；作废版本需及时回收
- 存档要求：验证文件原件（带签名）必须存档；电子文档需备份三份（本地+服务器+云端）；存档环境需防火防潮
- 保存期限：验证文件通常保存至设备退役后5年或产品退市后2年（以较长者为准）
- 常见不符合项：现场使用的验证方案版本与MFI记录不一致；测试记录签名不完整（缺签名、缺日期）；电子文档无审计追踪（Audit Trail）
